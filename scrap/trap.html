Preventing bot submissions

1) Putting input inside a very small transparent container

Input is perceivable by screen readers, so input instructions (not the label) should make clear that screen reader users should leave blank


    <label for="exampleFormControlInput1">Email address</label>
    <input
      type="email"
      id="exampleFormControlInput1"
      placeholder="name@example.com"
      aria-describedby="nofillinst">
    <p id="nofillinst">Do not fill this information</p>

Do not fill this information

2) Putting input in a container hidden via display:none

Will be hidden from screen reader users.

3) Hiding an input container with Javascript

Replacing the existing Qualtrics.SurveyEngine.addOnload to hide the input/label after document loads. Will be hidden from screen reader users. 


    Qualtrics.SurveyEngine.addOnload(function(){
      jQuery("#"+this.questionId).hide();
    });
