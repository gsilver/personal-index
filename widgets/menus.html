Guidelines


  • Keyboard
    • Trigger can be activated with Space or Enter keys
    • When trigger is activated and menu is open, the user can use the arrow up/down to navigate the menu choices (optimal) or the tab key to go down or shift+tab to go up
    • When component looses focus, menu dissapears if it was visible
  • Name, role, value
    • The trigger that opens the menu needs to be a button or have the button role and have a good name
    • Trigger will have aria-expanded=”true” when menu is visible, aria-expanded=”false” when not
    • After the menu loses focus, the menu closes (and the aria-expanded attribute of the trigger reflects this). Menus may lose focus when the user tabs off the last option or when the user activates any other control or clicks elsewhere.
    • The button may announce that it has a popup via aria-haspopup="true"
